# Research Summary

## When To Use
Use when the user asks for concise summaries of papers, RFCs, technical docs, or benchmark reports.

## Workflow
1. Identify the main question and scope.
2. Extract methods, assumptions, and limitations.
3. Summarize outcomes with concrete implications.
4. End with recommended next actions.

## Output Format
- `Key Point`: one-sentence core takeaway.
- `Evidence`: 3-5 short bullets.
- `Limitations`: 1-3 short bullets.
- `Next Steps`: numbered list.
